Task 3: SQL for Data Analysis

Objective

The objective of this task is to use SQL queries to extract, manipulate, and analyze data from a structured Ecommerce database. The task demonstrates the use of SQL for data analysis and database management.

Tools Used

* MySQL Workbench
* SQL

Database Used

A sample Ecommerce database consisting of the following tables:

* Customers
* Orders
* Products
* OrderDetails

Tasks Performed

1. Data Retrieval

* Used `SELECT` statements to retrieve data from tables.

2. Data Filtering

* Applied `WHERE` clauses to filter records based on specific conditions.

3. Data Sorting

* Used `ORDER BY` to sort query results.

4. Data Aggregation

* Applied aggregate functions such as:

  * `SUM()`
  * `AVG()`

5. Grouping Data

* Used `GROUP BY` to summarize customer spending.

6. Table Joins

* Implemented:

  * `INNER JOIN`
  * `LEFT JOIN`

7. Subqueries

* Created nested queries to identify customers with spending above the average order value.

8. Views

* Created a SQL View (`CustomerOrders`) for simplified analysis.

9. Indexing

* Created an index on `CustomerID` to improve query performance and demonstrate query optimization techniques.

Files Included

* `SQL_Queries.sql` – Complete SQL script including database creation, data insertion, and analysis queries.
* `Screenshots/` – Screenshots showing query execution and outputs.
* `README.md` – Project documentation.

Key SQL Concepts Demonstrated

* SELECT
* WHERE
* ORDER BY
* GROUP BY
* INNER JOIN
* LEFT JOIN
* Aggregate Functions
* Subqueries
* Views
* Indexes

Outcome

This project demonstrates the ability to work with relational databases, perform data analysis using SQL, create reusable database objects, and optimize query performance through indexing.
